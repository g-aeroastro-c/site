# Design File Instructions

This document provides guidelines and instructions for working with the design file created in Figma. The purpose of this file is to ensure consistency, clarity, and collaboration among team members during the design and development process.

## About the Design File
The Figma design file contains the visual representation of the project's user interface, including layouts, components, typography, and color schemes. It serves as a single source of truth for designers, developers, and stakeholders.

## How to Access the File
1. Open [the Design here](https://www.figma.com/design/ihKk7KS9me8ZY26lFVyoGe/GAAC-Website?node-id=0-1&t=zEyN753O1MNUt0Nn-1).
2. Log in with your credentials.
3. Navigate to the shared project folder.
4. Open the design file named **`[Insert File Name]`**.

## Key Sections in the File
- **Wireframes**: Initial low-fidelity layouts for the project, the three pages contains the responsive beahavior of the website across multiple frames.
- **UI Components**: Reusable elements like buttons, forms, and icons.
- **Prototypes**: Interactive flows for user testing.
- **Style Guide**: Fonts, colors, and spacing guidelines.

## Collaboration Guidelines
- Use comments to provide feedback directly on the design.
- Avoid making changes without prior discussion.
- Follow the naming conventions for layers and components.

## Exporting Assets
1. Select the required asset in Figma.
2. Use the export panel to choose the format (SVG).
3. Save the exported file to the appropriate project folder.
